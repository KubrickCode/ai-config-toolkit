# 워크플로우 규칙

## 문서 업데이트

- 주요 작업이나 필수 변경이 필요한 기능 변경 시 CLAUDE.md와 README.md 항상 업데이트
- 사소한 변경은 무시

## 실패 기반 문서화

- Claude가 같은 실수를 반복하면 CLAUDE.md에 명시적 금지 추가
- 실패로부터 학습하고 재발 방지를 위해 문서화

## Git 작업

- 브랜치 생성이나 커밋 자동 실행 금지
- 항상 사용자에게 수동 실행 요청

## 작업 시작 전

- 근거 없는 가정 피하기 - 중요한 세부사항 검증
- 파일 경로 추측 금지 - Glob/Grep으로 찾기
- API 계약이나 함수 시그니처 추측 금지 - 실제 코드 읽기
- 패턴 기반 합리적 추론은 허용
- 중요한 결정에 불확실할 때 사용자에게 질문

## 컨텍스트 수집

- 작업 시작 전 항상 컨텍스트 수집
- 관련 파일 먼저 읽기 (맹목적 작업 금지)
- 코드베이스의 기존 패턴 확인
- 프로젝트 컨벤션 검토 (명명, 구조 등)

## 스코프 관리

- 이슈 크기와 범위 정확히 평가
- 간단한 작업에 과도한 엔지니어링 피하기
- 구현과 문서화 모두에 적용
- 장황한 문서는 사람에게 리뷰 부담 유발

## 프로세스 정리

- 백그라운드 프로세스 정리: 사용 후 dev 서버, watcher 등 항상 종료
- 포트 충돌 방지

## 언어 컨벤션

- 생성되는 모든 콘텐츠(주석, 에러 메시지, 로그, 테스트 설명, 문서)에 프로젝트 언어 컨벤션 적용
- 기존 코드베이스에서 프로젝트 언어(한국어/영어 등) 감지
- 대화 언어 기준으로 언어 혼용 금지 - 항상 프로젝트 컨벤션 따르기
