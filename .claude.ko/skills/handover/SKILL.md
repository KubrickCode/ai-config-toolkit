# 대화 인수인계 요약

AI 에이전트 인수인계를 위한 종합적인 요약 생성: $ARGUMENTS

## 필수 출력

이 커맨드는 마크다운 파일을 생성하고 저장한다. 항상:

1. 대화 내용 분석
2. Write 도구로 파일 생성
3. 저장 위치: `./handoff-summary-YYYYMMDD-HHMMSS.md`

## 출력 템플릿

```markdown
# 프로젝트 인수인계 요약

생성일: [타임스탬프]

---

## 개요

프로젝트와 주요 목표에 대한 간략한 설명

## 사용자 컨텍스트

- 기술적 배경과 선호도
- 구체적인 요구사항과 제약사항
- 커뮤니케이션 스타일 선호도

## 원래 요구사항

- 초기 문제 정의
- 주요 목표와 성공 기준

## 완료된 작업

### 작업 1: [설명]

- 수행한 내용과 이 접근법을 선택한 이유
- 주요 코드 변경사항과 영향받은 파일

### 작업 2: [설명]

...

## 기술적 결정

### 결정 1: [주제]

- 고려한 옵션, 선택한 접근법, 근거

## 현재 상태

- 작동 중인 기능
- 알려진 이슈나 제한사항

## 대기 중인 작업

- [ ] 작업 1
- [ ] 작업 2

## 중요 경고사항

- 변경 사항 깨짐 방지를 위한 핵심 정보
- 보안 또는 성능 고려사항

## 다음 단계

작업 계속을 위한 권장 조치

## 관련 파일

- 파일 1: 용도와 최근 변경사항
- 파일 2: 용도와 최근 변경사항
```

## 작성 가이드라인

인수인계 요약은 환각 위험이 높다. 다음 제약 적용:

- **완료 상태 구분**: "완료" (검증됨), "시도" (문제 있을 수 있음), "논의" (구현 안 됨)
- **사용자의 정확한 표현 인용**: 선호도, 제약사항, 요구사항
- **대화에서 확인된 사실만 포함** -- 언급되지 않은 결정이나 요구사항을 추론하지 말 것
- **불확실성 표시**: 세부사항이 불확실하면 "[확인 필요]" 표기
- **사용자의 실제 선호도 기록** (예: "이모지 없음", "한국어 응답"), 추론된 것이 아닌

## 자기검증 (필수)

파일 저장 전 검증:

- [ ] 모든 "완료" 항목이 대화에서 증거가 있음
- [ ] 논의되거나 시도된 것만인 작업이 완료로 표시되지 않음
- [ ] 사용자 선호도가 추론이 아닌 명시된 선호도를 정확히 반영함
- [ ] 기술적 결정이 조작된 근거가 아닌 실제 대화의 근거를 포함함
- [ ] 대기 중인 작업이 사용자 요청과 에이전트 제안을 구분함
- [ ] 템플릿 섹션을 "채우기" 위해 정보를 조작하지 않음 -- 보고할 것이 없으면 비워둘 것
