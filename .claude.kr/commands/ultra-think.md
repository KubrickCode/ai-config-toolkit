# 심층 분석 및 문제 해결 모드

심층 분석 및 문제 해결 모드

## 지침

1. **초고속 사고 모드 초기화**
   - 향상된 분석적 사고 요청 확인
   - 깊이 있고 체계적인 추론을 위한 맥락 설정
   - 문제 공간을 포괄적으로 탐색할 준비

2. **문제 또는 질문 파싱**
   - $ARGUMENTS에서 핵심 과제 추출
   - 모든 이해관계자와 제약 조건 식별
   - 암묵적 요구사항과 숨겨진 복잡성 인식
   - 가정에 의문을 제기하고 미지의 요소 파악

3. **다차원 분석**
   여러 관점에서 문제에 접근:

   ### 기술적 관점
   - 기술적 실현 가능성과 제약 조건 분석
   - 확장성, 성능, 유지보수성 고려
   - 보안 영향 평가
   - 기술 부채와 미래 대비 평가

   ### 비즈니스 관점
   - 비즈니스 가치와 ROI 이해
   - 출시 시간 압박 고려
   - 경쟁 우위 평가
   - 위험 대 보상 트레이드오프 평가

   ### 사용자 관점
   - 사용자 요구사항과 불편 사항 분석
   - 사용성과 접근성 고려
   - 사용자 경험 영향 평가
   - 엣지 케이스와 사용자 여정 고려

   ### 시스템 관점
   - 시스템 전체 영향 고려
   - 통합 지점 분석
   - 의존성과 결합도 평가
   - 창발적 행동 고려

4. **다중 솔루션 생성**
   - 최소 3-5가지 다른 접근법 브레인스토밍
   - 각 접근법에 대해 고려:
     - 장단점
     - 구현 복잡도
     - 리소스 요구사항
     - 잠재적 위험
     - 장기적 영향
   - 전통적이고 창의적인 솔루션 포함
   - 하이브리드 접근법 고려

5. **심층 분석**
   가장 유망한 솔루션에 대해:
   - 상세한 구현 계획 작성
   - 잠재적 함정과 완화 전략 식별
   - 단계별 접근법과 MVP 고려
   - 2차 및 3차 효과 분석
   - 실패 모드와 복구 방안 검토

6. **교차 도메인 사고**
   - 다른 산업이나 도메인에서 유사점 도출
   - 다른 맥락의 디자인 패턴 적용
   - 생물학적 또는 자연 시스템 유추 고려
   - 기존 솔루션의 혁신적 조합 탐색

7. **도전 및 개선**
   - 각 솔루션에 대해 악마의 옹호자 역할
   - 약점과 사각지대 식별
   - "만약에" 시나리오 고려
   - 가정 스트레스 테스트
   - 의도하지 않은 결과 탐색

8. **통찰력 종합**
   - 모든 관점의 통찰력 결합
   - 주요 의사결정 요소 식별
   - 중요한 트레이드오프 강조
   - 혁신적 발견 요약
   - 문제 공간의 미묘한 관점 제시

9. **구조화된 권고사항 제공**
   명확한 구조로 결과 제시:

   ```

   ## 문제 분석
   - 핵심 과제
   - 주요 제약사항
   - 성공의 핵심 요소

   ## 솔루션 옵션

   ### 옵션 1: [이름]
   - 설명
   - 장단점
   - 구현 접근법
   - 위험 평가

   ### 옵션 2: [이름]

   [유사한 구조]

   ## 권장사항
   - 권장 접근법
   - 근거
   - 구현 로드맵
   - 성공 지표
   - 위험 완화 계획

   ## 대안적 관점
   - 반대 의견
   - 미래 고려사항
   - 추가 연구가 필요한 영역
   ```

10. **메타 분석**
    - 사고 프로세스 자체에 대한 성찰
    - 불확실성 영역 식별
    - 편향이나 한계 인정
    - 필요한 추가 전문지식 제안
    - 권장사항에 대한 신뢰도 수준 제공

## 사용 예시

```bash
# 아키텍처 결정
/ultra-think 마이크로서비스로 마이그레이션할지 모놀리스를 개선할지?

# 복잡한 문제 해결
/ultra-think 비용을 절감하면서 10배 트래픽을 처리하도록 시스템을 확장하려면?

# 전략적 기획
/ultra-think 차세대 플랫폼을 위해 어떤 기술 스택을 선택해야 하나?

# 디자인 과제
/ultra-think 하위 호환성을 유지하면서 개발자 친화적인 API로 개선하려면?
```

## 핵심 원칙

- **제1원리 사고**: 근본적 진실로 분해
- **시스템 사고**: 상호연결과 피드백 루프 고려
- **확률적 사고**: 불확실성과 범위로 작업
- **역전 사고**: 해야 할 것뿐 아니라 피해야 할 것 고려
- **2차 사고**: 결과의 결과 고려

## 출력 기대사항

- 포괄적 분석 (일반적으로 2-4페이지 분량의 통찰력)
- 트레이드오프가 포함된 여러 실행 가능한 솔루션
- 명확한 추론 체인
- 불확실성 인정
- 실행 가능한 권장사항
- 새로운 통찰력이나 관점
