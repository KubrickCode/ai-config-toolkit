# GitHub Pull Request 리뷰 도우미

제공된 URL의 Pull Request를 리뷰하고 미해결 리뷰 코멘트를 분석하여 종합적인 코드 리뷰 피드백을 제공합니다.

## PR 정보

**PR URL**: $1

GitHub CLI를 사용하여 PR 세부정보 가져오기:

- PR 메타데이터: !gh pr view "$1" --json number,title,author,state,isDraft,reviews,comments,files
- 미해결 리뷰 코멘트: !gh api "$(echo "$1" | sed -E 's#https://github.com/([^/]+)/([^/]+)/pull/([0-9]+).*#repos/\1/\2/pulls/\3/comments#')" --jq '.[] | select(.position != null) | {path, line: .position, body, user: .user.login, resolved: .resolved}'

## 작업 지침

1. **PR 세부정보 가져오기**
   - URL에서 owner, repo, PR 번호 추출
   - `gh pr view`를 사용하여 기본 PR 정보 가져오기
   - `gh api`를 사용하여 resolved 상태가 포함된 리뷰 코멘트 가져오기
   - 미해결 코멘트만 필터링 (`resolved: false` 또는 null)

2. **코드 변경사항 분석**
   - 각 미해결 코멘트에 대해 컨텍스트 파악
   - `gh pr diff`를 사용하여 관련 코드 섹션 가져오기
   - 코멘트에서 언급된 파일 및 라인 번호 식별

3. **리뷰 기준**

   프로젝트의 언어 및 프레임워크에 따라 `.claude/skills` 디렉토리의 적절한 가이드라인을 자동으로 감지하여 적용합니다.

   중점 검토 항목:
   - **코딩 스타일**: 일관성, 가독성, 네이밍 컨벤션
   - **아키텍처**: 디자인 패턴, 관심사 분리, 모듈화
   - **부작용**: 의도하지 않은 결과, 경쟁 조건, 상태 변경
   - **잠재적 버그**: 엣지 케이스, 에러 처리, null/undefined 체크

4. **리뷰 피드백 제공 및 수정 적용**

   각 미해결 코멘트에 대해:
   - **코멘트 컨텍스트**: 원본 리뷰 코멘트 인용
   - **파일 및 위치**: `파일경로:라인번호`
   - **코드 분석**: 관련 코드 섹션 검토
   - **평가**:
     - 우려사항이 타당한가?
     - 어떤 영향이 있는가?
     - 권장 솔루션은 무엇인가?
   - **결정**:
     - 우려사항이 타당하고 수정이 명확한 경우: **즉시 수정 적용** (Edit/Write 도구 사용)
     - 논의가 필요하거나 설계 결정이 필요한 경우: **사용자에게 질문** 후 변경
   - **대안적 접근법**: 해당되는 경우, 더 나은 구현 제안
   - **스킬 참조**: 수정 설명 시 스킬의 관련 가이드라인 인용

5. **요약**

   전체 평가 제공:
   - 리뷰한 미해결 코멘트 수
   - 자동으로 수정한 이슈 수
   - 논의가 필요한 이슈 (있는 경우)
   - 추가 주의가 필요한 남은 이슈
   - 전체 PR 품질 평가

## 출력 형식

```markdown
# PR 리뷰: [PR 제목]

**PR**: [PR URL]
**상태**: [열림/초안/닫힘]
**변경된 파일 수**: [개수]

## 미해결 리뷰 코멘트 분석 및 수정

### 1. [파일 경로]:[라인]

**@[사용자명]의 원본 코멘트**:

> [인용된 코멘트]

**분석**:
[리뷰 기준에 따른 상세 분석]

**적용된 조치**:

- ✅ **자동 수정 완료**: [적용된 수정 내용 설명]
- 또는
- ⏸️ **논의 필요**: [사용자 입력이 필요한 이유]

**관련 가이드라인**: [해당되는 경우 스킬 참조]

---

### 2. [파일 경로]:[라인]

...

## 전체 평가

- **리뷰한 코멘트**: [개수]
- **자동 수정된 이슈**: [개수]
- **논의가 필요한 이슈**: [개수]
- **전체 상태**: [모든 이슈 해결됨 / 부분 해결 / 사용자 입력 대기 중]

## 추가 의견

[PR 품질, 아키텍처 또는 사용된 패턴에 대한 기타 관찰사항]
```

## 중요 사항

- **비공개 저장소**: `gh`로 접근할 수 없는 경우, 시스템이 자동으로 GitHub MCP 도구를 사용 시도
- **해결된 코멘트**: 해결된 것으로 표시된 코멘트는 무시
- **집중 범위**: 전체 PR이 아닌 미해결 코멘트에서 제기된 이슈만 리뷰
- **스킬**: 프로젝트 언어/프레임워크를 자동 감지하고 적절한 스킬 참조
- **자동 수정**: 명확한 이슈는 질문 없이 자동으로 수정 적용
- **사용자 상담**: 다음의 경우에만 사용자에게 질문:
  - 아키텍처 또는 설계 결정이 필요한 경우
  - 여러 가지 유효한 접근법이 있는 경우
  - 잠재적인 Breaking Change가 있는 경우
  - 요구사항이 모호한 경우

## 사용 예시

```bash
/review-pr https://github.com/owner/repo/pull/123
```

이 커맨드는 PR #123의 모든 미해결 리뷰 코멘트를 분석하고 스킬 디렉토리의 적절한 코딩 가이드라인을 기반으로 상세한 피드백을 제공합니다.
